## Get Started Fast

This guide will walk you through setting up a fully functional Express server with database connectivity and creating your first API endpoint.

<Steps>
  <Step title="Clone and Install">
    First, clone the repository and install dependencies:

    <CodeGroup>
    ```bash npm
    git clone https://github.com/your-username/backend-template.git
    cd backend-template
    npm install
    ```

    ```bash yarn
    git clone https://github.com/your-username/backend-template.git
    cd backend-template
    yarn install
    ```

    ```bash pnpm
    git clone https://github.com/your-username/backend-template.git
    cd backend-template
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment">
    Create a `.env` file in the root directory with your database credentials:

    ```bash .env
    DATABASE_URL=postgres://postgres:root@127.0.0.1:5432/your_database_name
    PORT=8080
    ```

    <Note>
      Replace `your_database_name` with your PostgreSQL database name. Make sure PostgreSQL is running on your system.
    </Note>

    The template uses the `.env.example` file as a reference:

    ```bash
    cp .env.example .env
    # Then edit .env with your database credentials
    ```
  </Step>

  <Step title="Start the Server">
    Run the development server with hot-reload:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>

    You should see:

    ```bash
    DB connected
    👉 Server running on port 8080
    👉 Link http://localhost:8080
    ```

    <Note>
      The `dev` script uses Nodemon, which automatically restarts the server when you make code changes.
    </Note>
  </Step>

  <Step title="Test Your Server">
    Open your browser or use curl to test the welcome route:

    ```bash
    curl http://localhost:8080
    ```

    You should receive:

    ```
    Welcome to express!
    ```

    The server is now running! This route is defined in `src/app.js:19-21`:

    ```javascript src/app.js
    app.get('/', (req, res) => {
        return res.send("Welcome to express!");
    })
    ```
  </Step>

  <Step title="Create Your First Route">
    Let's create a simple API endpoint. Open `src/routes/index.js` and add:

    ```javascript src/routes/index.js
    const express = require('express');
    const router = express.Router();

    // Your first API route
    router.get('/api/hello', (req, res) => {
        res.json({
            message: 'Hello World!',
            timestamp: new Date().toISOString(),
            status: 'success'
        });
    });

    module.exports = router;
    ```

    Test your new endpoint:

    ```bash
    curl http://localhost:8080/api/hello
    ```

    Response:

    ```json
    {
      "message": "Hello World!",
      "timestamp": "2026-03-14T10:30:00.000Z",
      "status": "success"
    }
    ```
  </Step>
</Steps>

## Understanding the Code

### Application Architecture

The template separates concerns into distinct files:

<Tabs>
  <Tab title="server.js">
    **Entry Point** - Connects to the database and starts the server

    ```javascript src/server.js
    const app = require('./app');
    const sequelize = require('./utils/connection');

    const PORT = process.env.PORT || 8080;

    const main = async () => {
        try {
            sequelize.sync();
            console.log("DB connected");
            app.listen(PORT);
            console.log(`👉 Server running on port ${PORT}`);
            console.log(`👉 Link http://localhost:${PORT}`);
        } catch (error) {
            console.log(error)
        }
    }

    main();
    ```

    The `sequelize.sync()` call creates database tables based on your models.
  </Tab>

  <Tab title="app.js">
    **App Configuration** - Sets up Express middleware and routing

    ```javascript src/app.js
    const express = require('express');
    const helmet = require('helmet');
    const cors = require('cors');
    const router = require('./routes');
    const errorHandler = require('./utils/errorHandler');
    require('dotenv').config();

    const app = express();

    // Middlewares
    app.use(express.json());
    app.use(helmet({
        crossOriginResourcePolicy: false,
    }));
    app.use(cors());

    app.use(router);
    app.get('/', (req, res) => {
        return res.send("Welcome to express!");
    })

    // Error handling middleware (must be last)
    app.use(errorHandler)

    module.exports = app;
    ```
  </Tab>

  <Tab title="connection.js">
    **Database Setup** - Configures Sequelize with PostgreSQL

    ```javascript src/utils/connection.js
    const { Sequelize } = require('sequelize');
    require('dotenv').config();

    const sequelize = new Sequelize(process.env.DATABASE_URL, { 
        logging: false 
    })

    module.exports = sequelize;
    ```

    The `logging: false` option disables SQL query logging in the console.
  </Tab>
</Tabs>

## Error Handling

The template includes built-in error handling for common Sequelize errors:

```javascript src/utils/errorHandler.js
const errorHandler = (error, _req, res, _next) => {
    if(error.name === 'SequelizeValidationError') {
        const errObj = {};
        error.errors.map(er => {
            errObj[er.path] = er.message;
        })
        return res.status(400).json(errObj);
    }
    if(error.name === 'SequelizeForeignKeyConstraintError'){
        return res.status(400).json({ 
            message: error.message,
            error: error.parent.detail
        });
    }
    return res.status(500).json({
        message: error.message,
        error: error
    });
}
```

<Warning>
  For production environments, avoid exposing detailed error objects. Consider using environment-specific error responses.
</Warning>
