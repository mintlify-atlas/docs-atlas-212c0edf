## Overview

Middleware functions are the backbone of Express applications. They have access to the request object (`req`), response object (`res`), and the next middleware function in the application's request-response cycle.

## Built-in Middleware

The Backend Template includes several essential middleware configurations in `src/app.js`.

### JSON Body Parser

Parses incoming JSON request bodies (`src/app.js:12`):

```javascript src/app.js
app.use(express.json());
```

This allows you to access `req.body` in your routes:

```javascript
router.post('/users', async (req, res) => {
    const { name, email } = req.body; // Available thanks to express.json()
    // ...
});
```

### Helmet (Security)

Helmet helps secure Express apps by setting HTTP headers (`src/app.js:13-15`):

```javascript src/app.js
const helmet = require('helmet');

app.use(helmet({
    crossOriginResourcePolicy: false,
}));
```

<AccordionGroup>
  <Accordion title="What Helmet Does">
    Helmet sets security-related HTTP headers:

    - **X-DNS-Prefetch-Control:** Controls browser DNS prefetching
    - **X-Frame-Options:** Prevents clickjacking attacks
    - **X-Content-Type-Options:** Prevents MIME type sniffing
    - **X-XSS-Protection:** Enables XSS filter
    - **Strict-Transport-Security:** Enforces HTTPS
    - And many more...

    Configuration note: `crossOriginResourcePolicy: false` allows cross-origin resource loading (useful for CDN assets).
  </Accordion>

  <Accordion title="Custom Helmet Configuration">
    You can customize Helmet's behavior:

    ```javascript src/app.js
    app.use(helmet({
        crossOriginResourcePolicy: false,
        contentSecurityPolicy: {
            directives: {
                defaultSrc: ["'self'"],
                styleSrc: ["'self'", "'unsafe-inline'"],
                scriptSrc: ["'self'", "trusted-cdn.com"],
                imgSrc: ["'self'", "data:", "cloudinary.com"]
            }
        },
        hsts: {
            maxAge: 31536000,
            includeSubDomains: true,
            preload: true
        }
    }));
    ```
  </Accordion>
</AccordionGroup>

### CORS (Cross-Origin Resource Sharing)

Enables CORS for cross-origin requests (`src/app.js:16`):

```javascript src/app.js
const cors = require('cors');

app.use(cors());
```

This allows your API to be accessed from different domains (e.g., frontend applications).

<Note>
  The default CORS configuration allows all origins. For production, restrict this:

  ```javascript
  app.use(cors({
      origin: ['https://yourfrontend.com', 'https://admin.yourfrontend.com'],
      credentials: true
  }));
  ```
</Note>

## Middleware Execution Order

Middleware order matters! The template follows this pattern:

```javascript src/app.js
const app = express();

// 1. Built-in middleware (parsing, security)
app.use(express.json());
app.use(helmet({ crossOriginResourcePolicy: false }));
app.use(cors());

// 2. Custom middleware (authentication, logging, etc.)
// app.use(authMiddleware); // Add your custom middleware here

// 3. Routes
app.use(router);
app.get('/', (req, res) => {
    return res.send("Welcome to express!");
});

// 4. Error handlers (must be last!)
app.use(errorHandler);
```

<Warning>
  Error handling middleware must be registered **after** all routes and other middleware (`src/app.js:24`).
</Warning>

## Creating Custom Middleware

### Basic Middleware Structure

Middleware functions have three parameters:

```javascript
const myMiddleware = (req, res, next) => {
    // Do something with req/res
    console.log('Request received:', req.method, req.path);
    
    // Call next() to pass control to the next middleware
    next();
};

app.use(myMiddleware);
```

### Request Logger Middleware

Create a simple request logger:

```javascript src/middlewares/logger.js
const logger = (req, res, next) => {
    const timestamp = new Date().toISOString();
    console.log(`[${timestamp}] ${req.method} ${req.path}`);
    next();
};

module.exports = logger;
```

Register in `src/app.js`:

```javascript src/app.js
const logger = require('./middlewares/logger');

app.use(logger);
```

### Authentication Middleware

Protect routes with authentication:

```javascript src/middlewares/auth.js
const jwt = require('jsonwebtoken');

const verifyJWT = (req, res, next) => {
    // Get token from header
    const token = req.headers.authorization?.split(' ')[1]; // Bearer TOKEN
    
    if (!token) {
        return res.status(401).json({ message: 'No token provided' });
    }
    
    try {
        // Verify token
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        req.user = decoded; // Attach user info to request
        next();
    } catch (error) {
        return res.status(401).json({ message: 'Invalid token' });
    }
};

module.exports = verifyJWT;
```

Use in specific routes:

```javascript src/routes/post.routes.js
const verifyJWT = require('../middlewares/auth');

// Public route (no auth)
router.get('/', catchError(async (req, res) => {
    const posts = await Post.findAll();
    return res.json(posts);
}));

// Protected route (requires auth)
router.post('/', verifyJWT, catchError(async (req, res) => {
    const post = await Post.create({
        ...req.body,
        userId: req.user.id // From decoded JWT
    });
    return res.status(201).json(post);
}));
```

### Role-Based Authorization

Restrict routes by user role:

```javascript src/middlewares/authorize.js
const authorize = (...allowedRoles) => {
    return (req, res, next) => {
        if (!req.user) {
            return res.status(401).json({ message: 'Unauthorized' });
        }
        
        if (!allowedRoles.includes(req.user.role)) {
            return res.status(403).json({ 
                message: 'Forbidden: Insufficient permissions' 
            });
        }
        
        next();
    };
};

module.exports = authorize;
```

Usage:

```javascript src/routes/user.routes.js
const verifyJWT = require('../middlewares/auth');
const authorize = require('../middlewares/authorize');

// Only admins can delete users
router.delete('/:id', 
    verifyJWT, 
    authorize('admin'), 
    catchError(async (req, res) => {
        await User.destroy({ where: { id: req.params.id } });
        return res.status(204).send();
    })
);

// Both admins and moderators can update users
router.put('/:id', 
    verifyJWT, 
    authorize('admin', 'moderator'), 
    catchError(async (req, res) => {
        const user = await User.findByPk(req.params.id);
        await user.update(req.body);
        return res.json(user);
    })
);
```

### Validation Middleware

Validate request data before processing:

```javascript src/middlewares/validate.js
const validateUser = (req, res, next) => {
    const { email, password, firstName } = req.body;
    const errors = [];
    
    if (!email || !email.includes('@')) {
        errors.push('Valid email is required');
    }
    
    if (!password || password.length < 8) {
        errors.push('Password must be at least 8 characters');
    }
    
    if (!firstName) {
        errors.push('First name is required');
    }
    
    if (errors.length > 0) {
        return res.status(400).json({ errors });
    }
    
    next();
};

module.exports = { validateUser };
```

Apply to routes:

```javascript src/routes/user.routes.js
const { validateUser } = require('../middlewares/validate');

router.post('/', validateUser, catchError(async (req, res) => {
    const user = await User.create(req.body);
    return res.status(201).json(user);
}));
```

### Rate Limiting Middleware

Protect against abuse:

```javascript src/middlewares/rateLimiter.js
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
    windowMs: 15 * 60 * 1000, // 15 minutes
    max: 100, // Limit each IP to 100 requests per windowMs
    message: 'Too many requests, please try again later',
    standardHeaders: true,
    legacyHeaders: false,
});

const strictLimiter = rateLimit({
    windowMs: 15 * 60 * 1000,
    max: 5, // Only 5 requests per 15 minutes
    message: 'Too many login attempts, please try again later'
});

module.exports = { limiter, strictLimiter };
```

Apply globally or per-route:

```javascript src/app.js
const { limiter } = require('./middlewares/rateLimiter');

// Apply to all routes
app.use(limiter);
```

```javascript src/routes/auth.routes.js
const { strictLimiter } = require('../middlewares/rateLimiter');

// Apply only to login route
router.post('/login', strictLimiter, catchError(async (req, res) => {
    // Login logic
}));
```

## Error Handling Middleware

The template includes a centralized error handler (`src/utils/errorHandler.js:1-26`):

```javascript src/utils/errorHandler.js
const errorHandler = (error, _req, res, _next) => {
    // Sequelize validation errors
    if(error.name === 'SequelizeValidationError') {
        const errObj = {};
        error.errors.map(er => {
            errObj[er.path] = er.message;
        })
        return res.status(400).json(errObj);
    }
    
    // Foreign key constraint errors
    if(error.name === 'SequelizeForeignKeyConstraintError'){
        return res.status(400).json({ 
            message: error.message,
            error: error.parent.detail
        });
    }
    
    // Database errors
    if(error.name === 'SequelizeDatabaseError'){
        return res.status(400).json({ 
            message: error.message
        });
    }
    
    // Generic errors
    return res.status(500).json({
        message: error.message,
        error: error
    });
}

module.exports = errorHandler;
```

<Note>
  Error middleware must have **four** parameters `(error, req, res, next)` for Express to recognize it as error-handling middleware.
</Note>

## Middleware Best Practices

<AccordionGroup>
  <Accordion title="1. Always Call next()">
    If your middleware doesn't send a response, you must call `next()` to pass control:

    ```javascript
    const myMiddleware = (req, res, next) => {
        console.log('Processing...');
        next(); // Don't forget this!
    };
    ```
  </Accordion>

  <Accordion title="2. Handle Async Errors">
    Wrap async middleware to catch errors:

    ```javascript
    const asyncMiddleware = fn => (req, res, next) => {
        Promise.resolve(fn(req, res, next)).catch(next);
    };

    app.use(asyncMiddleware(async (req, res, next) => {
        await someAsyncOperation();
        next();
    }));
    ```
  </Accordion>

  <Accordion title="3. Order Matters">
    Register middleware in the correct order:
    1. Body parsers first
    2. Security middleware (Helmet, CORS)
    3. Custom middleware (logging, auth)
    4. Routes
    5. Error handlers last
  </Accordion>

  <Accordion title="4. Use Route-Specific Middleware">
    Don't apply middleware globally if only some routes need it:

    ```javascript
    // Bad: All routes require auth
    app.use(verifyJWT);
    app.use(router);

    // Good: Only specific routes require auth
    app.use(router);
    router.post('/protected', verifyJWT, handler);
    ```
  </Accordion>
</AccordionGroup>
